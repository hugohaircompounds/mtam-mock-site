# Requirements: MTAM Homepage Demo — JTBD Redesign

**Defined:** 2026-03-30
**Core Value:** Every section of the page exists because a JTBD framework decision put it there — the demo must visually prove that AI + JTBD produces intentional, customer-job-driven design.

## v1 Requirements

### Page Structure

- [ ] **STRC-01**: Page sections follow JTBD Job Map order: Define > Locate > Prepare > Confirm > Execute
- [ ] **STRC-02**: Single self-contained HTML file with all CSS/JS inline (CDN-loaded fonts/icons only)
- [ ] **STRC-03**: File saved to `slideshows/` directory
- [ ] **STRC-04**: Desktop-first layout optimized for presentation screen viewing

### Hero / Define Zone

- [ ] **HERO-01**: Hero section with outcome-based headline addressing the functional job
- [ ] **HERO-02**: Subheadline addressing the emotional job (confidence, no judgment)
- [ ] **HERO-03**: Primary CTA button above the fold
- [ ] **HERO-04**: Problem-first framing section naming specific stylist struggles (Struggle Stack)
- [ ] **HERO-05**: Anti-gatekeeping manifesto block addressing identity/social job

### Content / Locate Zone

- [ ] **CONT-01**: Course topics list using outcome language (not feature counts)
- [ ] **CONT-02**: Guest educator grid with name + specialization label + placeholder image
- [ ] **CONT-03**: Founder identity section with struggle-origin narrative

### Trust / Prepare + Confirm Zone

- [ ] **TRST-01**: Outcome-based testimonials structured around practice changes (CSS-modeled)
- [ ] **TRST-02**: Pricing section with 3 tiers (monthly/quarterly/annual)
- [ ] **TRST-03**: Value comparison framing anchored against per-method certification costs

### Conversion / Execute Zone

- [ ] **CONV-01**: Final CTA section with job-completion language ("Start learning" not "Subscribe")
- [ ] **CONV-02**: Footer with minimal navigation (Terms, Contact, Copyright)

### Visual Design

- [ ] **DSGN-01**: Polished, presentation-ready visual design with modern typography
- [ ] **DSGN-02**: MTAM brand essence retained (professional, approachable, empowerment-focused)
- [ ] **DSGN-03**: Smooth scroll behavior and visual hierarchy supporting job progression
- [ ] **DSGN-04**: Black/white base aesthetic with warm accent color

### Deliverables

- [ ] **DLVR-01**: Prompt list document in bullet format showing prompts used
- [ ] **DLVR-02**: JTBD decision notes mapping each section to the framework decision that shaped it

## Out of Scope

| Feature | Reason |
|---------|--------|
| Backend / database | Single static HTML file demo |
| Working authentication | Demo-only, no user accounts |
| Multiple pages / routing | Homepage only for presentation |
| Presentation slide (PPT/Slides) | Handled separately outside this project |
| E-commerce / payment integration | No real transactions |
| Mobile-responsive perfection | Desktop-first for presentation screen |
| Video embeds | External dependency risk during presentation |
| Live chat / contact widget | Third-party script dependencies |
| Countdown timers / urgency elements | Contradicts anti-gatekeeping brand voice |
| JTBD annotations on the page itself | Keep page clean; annotations go in decision notes doc |
