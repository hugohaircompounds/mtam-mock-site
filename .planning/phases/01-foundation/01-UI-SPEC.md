### CDN Load Order

Load scripts in this exact order inside `<head>`. Do not reorder — Tailwind must compile before content renders.

```html
<!-- Fonts (preconnect first, then stylesheet) -->
<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=DM+Serif+Display:ital@0;1&family=Manrope:wght@400;600&display=swap" rel="stylesheet">

<!-- Icons: Phosphor (regular + fill weights only) -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@phosphor-icons/web@2.1.2/src/regular/style.css">
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/@phosphor-icons/web@2.1.2/src/fill/style.css">

<!-- Tailwind v4 Play CDN -->
<script src="https://cdn.jsdelivr.net/npm/@tailwindcss/browser@4"></script>

<!-- GSAP Core + ScrollTrigger -->
<script src="https://cdn.jsdelivr.net/npm/gsap@3.14.2/dist/gsap.min.js"></script>
<script src="https://cdn.jsdelivr.net/npm/gsap@3.14.2/dist/ScrollTrigger.min.js"></script>
```

### Design Token Block

All CSS custom properties defined once at `:root`. Every component, section, and utility references these — never hardcodes hex values directly. Changing one token updates the whole page (ROADMAP success criterion 2).

```css
:root {
  /* Surfaces */
  --color-bg:           #0f0f0f;   /* near-black — primary page background */
  --color-surface:      #1a1a1a;   /* cards, sections, nav bar */
  --color-border:       #2a2a2a;   /* subtle borders between elements */

  /* Text */
  --color-text-primary: #f5f5f5;   /* near-white — headlines, body copy */
  --color-text-muted:   #9ca3af;   /* gray-400 — captions, metadata, labels */

  /* Accent */
  --color-accent:       #C9A84C;   /* warm gold — CTAs, highlights, dividers */
  --color-accent-hover: #b8953e;   /* darkened gold for hover state on CTAs */

  /* Semantic */
  --color-error:        #ef4444;   /* red-500 — error states only (Phase 2+) */

  /* Typography */
  --font-display: 'DM Serif Display', serif;
  --font-body:    'Manrope', sans-serif;

  /* Elevation */
  --radius-card:  16px;
  --shadow-card:  0 4px 32px rgba(0, 0, 0, 0.4);
  --shadow-deep:  0 8px 48px rgba(0, 0, 0, 0.6);
}
```

### Tailwind Theme Extension

Extend Tailwind's default theme via a `<style type="text/tailwindcss">` block immediately after the Tailwind CDN script tag. This registers brand tokens as Tailwind utilities so they can be used as `bg-brand-bg`, `text-brand-accent`, etc.

```html
<style type="text/tailwindcss">
  @theme {
    --color-brand-bg:           #0f0f0f;
    --color-brand-surface:      #1a1a1a;
    --color-brand-border:       #2a2a2a;
    --color-brand-text:         #f5f5f5;
    --color-brand-muted:        #9ca3af;
    --color-brand-accent:       #C9A84C;
    --color-brand-accent-hover: #b8953e;

    --font-display: 'DM Serif Display', serif;
    --font-body:    'Manrope', sans-serif;

    --radius-card: 16px;
  }
</style>
```

| Token | Value | Usage |
|-------|-------|-------|

## Typography

| Role | Font | Size | Weight | Line Height | Usage |
|------|------|------|--------|-------------|-------|
| Display | DM Serif Display | 56px | 400 (serif italic as decorative variant) | 1.1 | Hero headline only |
| Heading | DM Serif Display | 36px | 400 | 1.2 | Section headings (h2) |
| Subheading / CTA | Manrope | 16px | 600 | 1.3 | Section subheadings, card titles (h3), button copy |
| Body / Label | Manrope | 16px / 14px | 400 | 1.6 / 1.4 | Paragraph text (16px); captions, metadata, tag labels (14px) |

**Declared sizes:** 14px, 16px, 36px, 56px — exactly 4.

**Declared weights:** 400 (regular) and 600 (semibold) — exactly 2. No other weights are used in the type hierarchy.

**Subheading vs Body distinction:** Both Subheading and Body share 16px. They are differentiated by weight alone — Subheading uses weight 600, Body uses weight 400. This is sufficient visual distinction; no additional size or color difference is needed.

**Label vs Body distinction:** Label uses 14px at weight 400 and `--color-text-muted` (#9ca3af). The smaller size and muted color provide sufficient distinction from Body (16px at weight 400 on `--color-text-primary`) without introducing a third weight.

**Size rule:** 4 declared sizes only: 14, 16, 36, 56. No other sizes permitted.

**Weight rule:** 2 declared weights only: 400 and 600. Weight 700 is not used. Weight 500 is not used.

## Color

| Role | Hex | % | Usage |
|------|-----|---|-------|
| Dominant surface | `#0f0f0f` | 60% | Page background — `<body>`, full-bleed sections |
| Secondary surface | `#1a1a1a` | 30% | Cards, nav bar, section alternates, testimonial blocks |
| Accent | `#C9A84C` | 10% | See reserved list below |
| Text primary | `#f5f5f5` | — | All headline and body copy on dark backgrounds |
| Text muted | `#9ca3af` | — | Captions, metadata, placeholder labels |
| Border | `#2a2a2a` | — | Subtle dividers between elements |
| Destructive | `#ef4444` | — | Error states only (no destructive actions in Phase 1) |

**Accent (`#C9A84C`) reserved for:**
1. Primary CTA button background
2. CTA button hover border glow
3. Section divider accent lines (thin horizontal rules between major sections)
4. Active/hover state on navigation links
5. Decorative pull-quote quotation marks
6. Star/rating icons in testimonials (Phase 2)

**Accent is NOT used for:** body text, card backgrounds, general borders, secondary buttons, hover states on non-CTA elements.

**Dark background rationale:** Near-black background maximizes contrast on a presentation projector, supports glassmorphism card effects, and matches MTAM's existing black/white brand.

## Layout Baseline

Phase 1 establishes the desktop layout container — the shell that all Phase 2 sections fill.

| Property | Value | Rationale |
|----------|-------|-----------|
| Max content width | 1200px | Presentation-screen-safe, no ultra-wide stretching |
| Content container | `max-w-[1200px] mx-auto px-8` | Centered with 32px side gutters |
| Base font size | 16px on `<html>` | `rem` units resolve cleanly |
| Scrollbar | Default browser scrollbar | Do not hide — presenter needs to scroll visibly |
| Overflow | `overflow-x: hidden` on `<body>` | Prevent horizontal scroll on desktop |
| Desktop target resolution | 1440 x 900px minimum | Standard presentation laptop |
