**Structure and content rules (per locked decisions D-01 through D-04):**

1. **Title:** Use a welcoming title like "How to Build Your Salon Website with AI — A Prompt Guide"

2. **Introduction (2-3 paragraphs):**
   - Explain this is a guide for using AI (specifically Claude) to build a professional salon website
   - Set the tone: conversational, like talking to a friend (per D-03). Example voice: "You don't need to know code. You just need to know what you want your site to feel like."
   - Mention this guide was created alongside a real demo to prove AI + intentional design thinking produces professional results

3. **Prompt sections — organized by DESIGN CONCEPT, not build phase (per D-02).** Foundation and Documentation phases are excluded entirely. Only include prompts related to designing the visible page. Suggested groupings:

   **a. "Start with your story" (struggle/manifesto prompts)**
   - Prompt about naming the specific frustrations your clients face before offering solutions
   - Prompt about writing an anti-gatekeeping or values statement that establishes your salon's philosophy
   - Example prompt format: "Tell Claude: 'Write me a section that names the top 5 frustrations hair stylists face when trying to learn new techniques. Use second-person language — speak directly to the reader. Don't mention my salon name yet.'"

   **b. "Make your first impression count" (hero section prompts)**
   - Prompt about writing an outcome-based headline (what the visitor achieves, not what you offer)
   - Prompt about writing a subheadline that addresses how the visitor wants to feel
   - Prompt about creating a call-to-action button with action language
   - Example: "Tell Claude: 'Write a hero headline for your salon site. Instead of saying what you offer, say what the client achieves. Bad: We offer 50+ extension techniques. Good: Master the techniques that make clients rebook.'"

   **c. "Show what you teach" (course topics / services prompts)**
   - Prompt about listing services or course topics using outcome language instead of feature lists
   - Prompt about creating visual cards with icons for each topic
   - Example: "Tell Claude: 'Create 6 service cards for your salon. Each card should have an icon, a short title, and one sentence about what the client walks away with — not what the service includes.'"

   **d. "Introduce your team" (educators/team prompts)**
   - Prompt about creating team member cards with specialization labels
   - Prompt about writing a founders section that tells the origin story through struggle, not credentials

   **e. "Build trust with real results" (testimonials prompts)**
   - Prompt about writing testimonials that describe specific practice changes, not generic praise
   - Example: "Tell Claude: 'Write 3 testimonials for your salon. Each one should describe a specific thing the client changed in their routine after visiting — not just say you're great.'"

   **f. "Price with confidence" (pricing prompts)**
   - Prompt about creating tiered pricing that anchors value against alternatives
   - Prompt about using a "Most Popular" badge to guide choice

   **g. "Close the deal" (CTA + footer prompts)**
   - Prompt about writing a final call-to-action that uses job-completion language ("Start learning" not "Subscribe")
   - Prompt about keeping the footer minimal (just essentials)

4. **Each prompt entry contains:**
   - A one-sentence setup explaining the design intent (why this prompt matters)
   - The actual prompt text in a blockquote, written conversationally as "Tell Claude: '...'" (per D-03)
   - Generalized for any hair salon — use "your salon" / "your services" / "your team" instead of MTAM-specific references (per D-04)

5. **Closing section:**
   - Brief paragraph encouraging the reader to iterate — AI works best as a conversation, not a single prompt
   - Mention they can adjust tone, length, and style by telling Claude what they want changed

**Tone:** Conversational, encouraging, zero jargon. Written like you're walking a friend through the process. Use "you" and "your" throughout. No code snippets or technical terms.

**Length:** Approximately 15-25 prompt entries total across all sections. Enough to be genuinely useful without being exhausting.

**Format:** Clean Markdown with `##` section headers, `>` blockquotes for prompts, short paragraphs between prompts for context.

Naming 5 specific stylist frustrations in second-person ('You learned a technique at a $2,000 workshop...') validates the visitor's experience before any solution is offered. This is struggle identification — the visitor thinks 'they understand my problem.'"

**Row 3: Manifesto (id="manifesto")**
- JTBD Stage: Define
- Framework: Four-Dimensional Job Analysis — emotional + social job dimensions (Marketing Frameworks #17) + JTBD Anti-Vanity Framework (Design Team #1)
- Why: "The manifesto addresses the emotional job (feel confident learning without judgment) and the social/identity job (be part of a community that rejects gatekeeping). Anti-Vanity ensures this section does real work — it bridges identity frustration to MTAM's philosophy rather than being decorative."

- Framework: Outcome-Based Messaging Strategy (Marketing Frameworks #18) + JTBD for IA — Top Task Analysis (Design Team #3, #4)

Value comparison anchoring ($249/year vs $500-$2,500 per single certification) reframes cost as an opportunity score: high importance (learn techniques) + low current satisfaction (overpriced alternatives) = high opportunity. 'Most Popular' badge reduces decision friction."

**Row 9: Final CTA (id="cta")**

- Why: "Execute stage: the customer performs the action that delivers core value. 'Start Learning Today' uses job-completion language — it describes the action the customer takes, not the platform's action. Success Language converts 'Subscribe Now' (platform verb) into 'Start Learning' (customer verb)."

**Row 10: Footer (id="footer")**
